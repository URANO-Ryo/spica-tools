setup_gns
===============
Usage   
--------
.. parsed-literal::

    cg_spica setup_gns [-Go] <:strong:`topfile 1`> <:strong:`nmol 1`> [<:strong:`topfile 2`> <:strong:`nmol 2`> ... <:strong:`topfile n`> <:strong:`nmol n`>] 
                       <:strong:`paramfile`> <:strong:`coordfile`>

Description
-----------
``setup_gns`` generates simulation input files for CG-MD with SPICA using GENESIS.
This program requires the following files as inputs:

- ``topfile``: SPICA topology file, distributed at `SPICA web <https://www.spica-ff.org/download.html>`_
- ``paramfile``: SPICA parameter file, distributed at `SPICA web <https://www.spica-ff.org/download.html>`_
- ``coordfile``: CG configuration file in PDB format

The ``topfile`` can be generated with ``json2top``, ``ENM``, or ``Go`` commands (for proteins).
For more information on these, see :doc:`json2top <json2top>`, :doc:`ENM <ENM>`, and :doc:`Go <Go>`.
The ``nmol`` is the number of ``topfile`` molecules included in your system.  
It is sensitive to the order of arguments, so the molecule topology and number 
should be given in the same order as given in the prepared configuration file in the PDB file. 

Running the program with the appropriate input files yields the following outputs:

- ``toppar/top_spica_charmm.rtf``: CHARMM-style RTF (residue topology file) for GENESIS
- ``toppar/par_spica_charmm.prm``: CHARMM-style parameter file for GENESIS
- ``gns_spica.psf``: PSF (protein structure file) containing system topology
- ``toppar/enm_bond_index.ndx``: ENM bond index file (only when -Go option is used)
- ``toppar/enm_bond_parm.prm``: ENM bond parameter file (only when -Go option is used)
- ``toppar/enm_angle_index.ndx``: ENM angle index file (only when -Go option is used)
- ``toppar/enm_angle_parm.prm``: ENM angle parameter file (only when -Go option is used)

The RTF and parameter files in the ``toppar/`` directory and the PSF file are required to run 
GENESIS CG-MD with SPICA FF. Use these files with the control file generated by ``gen_gnsin`` 
to start the simulation. For systems with Go model proteins (``-Go`` option), ENM files are 
automatically generated and should be included in the GENESIS control file.

Example
-------
**Lipid membrane system (no proteins):**

.. code-block:: bash

    cg_spica setup_gns DOPC.top 128 WAT.top 1408 spica_db.prm dopc.cg.pdb

:download:`DOPC.top <data/DOPC.top>`  
:download:`WAT.top <data/WAT.top>`  
:download:`spica_db.prm <data/spica_db.prm>`  
:download:`dopc.cg.pdb <data/dopc.cg.pdb>`  

Output files:

:download:`toppar/top_spica_charmm.rtf <data/top_spica_charmm.rtf>`  
:download:`toppar/par_spica_charmm.prm <data/par_spica_charmm.prm>`  
:download:`gns_spica.psf <data/gns_spica.psf>`  

**Protein wit ENM:**

.. code-block:: bash

    cg_spica setup_gns  protein.top 1 WAT.top 5000 spica_db.prm protein.cg.pdb

 This generates ENM files in addition to the standard output files:

- ``toppar/enm_bond_index.ndx``
- ``toppar/enm_bond_parm.prm``
- ``toppar/enm_angle_index.ndx``
- ``toppar/enm_angle_parm.prm``

**Multiple protein copies with ENM:**

.. code-block:: bash

    cg_spica setup_gns protein.top 3 WAT.top 5000 spica_db.prm system.pdb




The ``-Go`` option treats each protein as an independent topology. But, currently, GENESIS is not implemented this Go SPICA model.

Positional args
---------------

``topfile`` <topology>
    topology file of a molecule
``nmol`` <int>
    number of molecules
``paramfile`` <parameter>
    SPICA force field parameter file
``coordfile`` <.pdb>
    CG configuration file

Optional args
-------------

``-Go``
    Enable special handling for proteins with Go model or ENM. When this option is used, 
    each protein molecule is treated as an independent topology entry, even when multiple 
    copies (nmol > 1) are specified. This is required for correct indexing of Go/ENM 
    interactions within each protein molecule.

Important Notes
---------------

**File Order**
    The order of topology files and molecule numbers must match the order of molecules 
    in the PDB file.

**Multiple Molecules**
    When specifying multiple molecule types, provide them in pairs: 
    ``topfile1 nmol1 topfile2 nmol2 ... paramfile pdbfile``

**When to Use -Go Option**
    Use ``-Go`` when your topology file was generated by the ``Go`` or ``ENM`` commands 
    and contains proteins with Go-like model or elastic network constraints. This ensures 
    that each protein copy is treated independently for correct interaction indexing.
		GENESIS is not implemented this Go SPICA model.
    
**ENM File Detection**
    When topology files contain ENM or Go model parameters, the program automatically 
    generates ENM parameter files (``enm_bond_index.ndx``, ``enm_bond_parm.prm``, 
    ``enm_angle_index.ndx``, ``enm_angle_parm.prm``) in the ``toppar/`` directory. 
    These files are automatically detected by ``gen_gnsin``.

Complete Workflow
-----------------

**Step 1: Generate topology and parameter files**

.. code-block:: bash

    cg_spica setup_gns DOPC.top 128 WAT.top 1408 spica_db.prm system.pdb

**Step 2: Generate GENESIS control file**

.. code-block:: bash

    cg_spica gen_gnsin -pdb system.pdb

**Step 3: Run GENESIS simulation**

.. code-block:: bash

    export OMP_NUM_THREADS=4
    mpirun -n 8 spdyn npt.inp > npt.log
